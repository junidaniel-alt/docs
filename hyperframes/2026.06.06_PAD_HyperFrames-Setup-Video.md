# HyperFrames — Setup e Render de Vídeo Alta Resolução

## Retomada rápida
- **O que é?** HyperFrames (HeyGen, open-source, Apache 2.0): framework HTML → vídeo. Escreve-se HTML/CSS/JS e ele renderiza vídeo determinístico (MP4/MOV/WebM/PNG-seq) via Chrome headless + FFmpeg. É o "Remotion em HTML puro", feito para agentes (Claude Code) escreverem o código.
- **Onde parou?** Instalado e **verificado de ponta a ponta** no ambiente Claude Code web em 06/06/2026: renders 1080p e **4K (3840×2160)** H.264/30fps concluídos com sucesso.
- **Risco atual?** O container do Claude Code web é efêmero — o que se instala (FFmpeg, Chromium, npm) some quando ele é reciclado. Mitigação: script idempotente versionado no repo.
- **Próximo passo?** Em qualquer projeto que exija vídeo, rodar `setup-hyperframes.sh` no início e usar o fluxo abaixo.
- **Quem participa?** Daniel + Claude Code.
- **Prazo?** Sem prazo; capacidade permanente.

## Núcleo — como funciona
- **Autoria:** `index.html` define a composição com `data-width`/`data-height` (ex.: 1920×1080). Cenas/clipes são elementos com `data-start` e `data-duration`. Animação via timeline GSAP exposta em `window.__timelines["main"]`.
- **Requisitos:** Node ≥ 22 · FFmpeg · Chromium headless (baixado automaticamente pelo Puppeteer no 1º render, cache em `~/.cache`).
- **Saída determinística:** mesma entrada → mesmo vídeo, pixel-perfect.

## Fluxo de trabalho
```bash
npx hyperframes init meu-video
cd meu-video
npm run dev      # preview live-reload no browser
npm run check    # lint + validate + inspect
npm run render   # MP4 em renders/
```

## Alta resolução / melhor qualidade — flags do `render`
- `--resolution 4k` → 3840×2160 (também `portrait-4k`, `square-4k`, `1080p`); supersampling via deviceScaleFactor, a composição não muda.
- `--quality high` → `draft | standard | high`.
- `--fps 60` → 24/25/30/50/60/120/240 ou racional NTSC (`30000/1001`).
- `--format mov | webm` → canal **alfa** (transparência); `png-sequence` → frames RGBA p/ After Effects/Nuke/Fusion.
- `--crf 16` **ou** `--video-bitrate 40M` (mutuamente exclusivos) → controle fino.
- `--gpu` / `--browser-gpu` → encode/captura por GPU quando disponível. `--hdr`/`--sdr` força o modo.

Render topo de linha:
```bash
npx hyperframes render --resolution 4k --quality high --fps 60 \
  --video-bitrate 40M -o renders/final_4k.mp4
```

## Armadilhas confirmadas no ambiente
- **CDN + TLS:** o proxy do sandbox quebra `https` externo dentro do Chrome (`ERR_CERT_AUTHORITY_INVALID`). Libs por CDN (ex.: GSAP via jsDelivr) falham em silêncio e a animação não roda. **Vendore localmente:** `npm i gsap && cp node_modules/gsap/dist/gsap.min.js ./gsap.min.js` e referencie `./gsap.min.js`.
- **apt quebrado:** PPAs de terceiros (deadsnakes/ondrej) derrubam o `apt update` com erro de assinatura/403; desativá-los antes de instalar o FFmpeg resolve (o script já faz isso).
- **RAM:** ~256 MB por worker de render; em máquina ≤ 8 GB o HyperFrames entra em low-memory-mode automático.
