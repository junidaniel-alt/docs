# Incidente Crítico — Perda e Recuperação da PAD_Aprendizados

> Cópia versionada em git (redundância — regra 7 do Protocolo Anti-Perda). Original no cofre em `11_PADROES_OPERACAO`.

## Linha do tempo
- 04/06 20:21 — v2.0: 19.005 B (íntegra)
- 05/06 02:14 — v3.0: 21.086 B (pico; restaurada)
- 05/06 02:16 — v4.0: 12.723 B (perda de ~8 KB)
- 05/06 02:20 — v5.0: 321 B (truncamento total / stub)
- 06/06 — detectado, diagnosticado por histórico de versões, restaurada v3.0 → v6.0 (21.086 B), validada.

## Causa-raiz (5 porquês)
1. Ficou zerada → sobrescrita por stub de 321 B.
2. Sobrescrita → operação de edição reescreveu o arquivo inteiro em vez de acrescentar.
3. Reescreveu o inteiro → o cofre não tem append nativo; toda edição é reescrita integral, e operações grandes podem truncar.
4. Passou ~1 dia despercebido → sem validação pós-escrita; "sucesso de API" ≠ conteúdo íntegro.
5. Sem salvaguarda → não havia protocolo de integridade; só o histórico de versões salvou.

## Fragilidades
- F1 Edição = reescrita integral (casa com A10/A20 da própria nota).
- F2 "Sucesso de API" não garante integridade.
- F3 Sem validação pós-escrita.
- F4 Operações > ~15 KB via conector podem cair no meio (truncamento).
- F5 Detecção tardia (sem monitorar tamanho/diff).

## PROTOCOLO ANTI-PERDA (regra permanente)
1. Antes de editar: ler e guardar o conteúdo íntegro + anotar byte-count.
2. Preferir restaurar (restoreVersion) a reescrever — atômico e não-destrutivo.
3. Nunca reescrever nota grande (> ~15 KB) numa única operação.
4. Validar SEMPRE pós-escrita (byte-count ≥ esperado + seções-âncora presentes).
5. Suspeita de perda = parar; GET /versions e restaurar — nunca reescrever por cima.
6. Append de entrada: capturar atual → acrescentar → validar crescimento → pente-fino (A10).
7. Redundância: versionar notas-chave no git, fora do OneDrive.
8. Higiene: Create Text File gera "Nome 1" (não usar para editar); gravar em subpasta via itemId direto.

## Lições da sessão (A21–A27, para incorporar na PAD por append seguro)
- A21 Ambiente Claude Code Web é efêmero (instalações descartáveis; persistência = git+cofre).
- A22 Rede sandbox quebra TLS de CDNs no Chrome (vendorar libs); apt com PPAs quebrados.
